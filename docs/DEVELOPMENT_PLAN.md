# CrabDesk 开发计划

## 1. 产品定位

CrabDesk 是面向 Windows 10/11 x64 的桌面整理工具。产品以桌面盒子、真实桌面文件管理和自动整理为核心，功能目标参考成熟桌面整理软件，但不复制其品牌、素材、授权体系或界面布局。

必须长期保持以下产品原则：

- 盒子嵌入 Explorer 桌面层，不使用全局置顶窗口。
- `Win+D`、`Win+M` 和任务视图后盒子仍然可见，普通窗口可以正常覆盖盒子。
- 未分组项目保持 Explorer 原生显示、拖拽和菜单交互；CrabDesk 仅绘制盒子及盒内项目，暂停或退出后恢复分组项原来的 Explorer 位置。
- 项目放入盒子后不复制文件、不创建真实子文件夹，也不改变原始路径。
- 设置页中的每个选项都必须连接真实行为，不提供无效占位开关。
- 默认配置以可靠、克制为先；透明度、自动整理等可能改变桌面行为的功能必须由用户主动开启。
- 首次启动保持零盒子并暂停桌面接管；用户从托盘菜单或设置页执行“智能整理”后，仅为实际匹配规则的项目创建盒子，使用避让式网格布局防止重叠，再显式启用接管。

## 3. 设置中心结构

最终设置中心规划为以下页面：

### 3.1 常规

- 界面语言，首版保留简体中文，后续增加英文资源。
- 登录 Windows 后启动。
- 启动后仅驻留托盘。
- 启动时确保 Explorer 桌面图标可见。
- 双击桌面空白区域显示或隐藏图标。
- 鼠标移入标题栏时展开折叠盒子。
- 文件重命名或桌面项目变化后刷新分组。
- 深色、浅色、跟随系统主题切换。

### 3.2 整理规则

- 总开关：启用自动整理。
- 触发方式：程序启动、桌面实时变化、手动立即整理。
- 规则条件：项目类型、扩展名、名称通配符、目录或快捷方式。
- 规则动作：分配到指定盒子、保持未分组或忽略。
- 规则优先级：按列表顺序匹配，首条命中规则生效。
- 内置规则：快捷方式、目录、文档、图片、压缩包。
- 自定义规则的新增、复制、启用、禁用、排序和删除。
- 预览本次整理结果，确认后再批量写入分组元数据。

自动整理只改变 CrabDesk 分组，不移动真实文件。默认关闭实时整理，避免首次运行改变用户桌面。

### 3.3 主题与外观

分为盒子、图标和交互三个区域：

- 选择全部盒子或单个盒子进行配置。
- 盒子背景色、标题色、强调色、边框和圆角。
- 盒子透明度，默认保持 `100%` 不透明。
- 标题栏高度、标题位置和折叠按钮显示。
- 图标尺寸、横向间距、纵向间距和标签字体大小。
- 网格或列表视图，以及名称、类型、修改时间和手动排序。
- 是否仅显示图标、是否显示快捷方式角标。
- 选中区域颜色和悬停反馈。
- 所有外观修改提供实时预览和恢复默认值。

### 3.4 布局备份

- 手动创建布局备份。
- 导出和导入配置文件。
- 每日自动备份以及保留天数设置。
- 显示备份时间、显示器信息和盒子数量。
- 恢复前自动保存当前布局，恢复失败时回滚。
- 分辨率或显示器缺失时按显示器标识迁移并限制到可见区域。

备份只包含布局、规则和设置，不备份桌面文件内容。

### 3.5 特殊盒子

- 文件夹映射盒子，显示指定目录内容但不移动文件。
- 支持只读目录和可写目录。
- 文件系统变化实时刷新。
- 映射目录不存在或磁盘离线时显示明确状态。
- 后续评估多标签组合盒子，不作为首个实现目标。

### 3.6 高级

- 显示桌面快捷键和整理桌面快捷键。
- 盒子移动、折叠和主题切换动画。
- 托盘菜单提供创建盒子、智能整理和打开设置入口，不修改 Explorer 桌面右键菜单。
- 是否显示系统 Shell 项目。
- 动态壁纸兼容模式。
- Explorer 重连和桌面宿主诊断信息。
- 清理缩略图缓存和重置布局。

### 3.7 检查更新

检查更新统一依托 GitHub Releases，不建设独立更新服务器：

- 手动“检查更新”始终可用。
- 可选“启动时检查更新”，默认开启但不阻塞 CrabDesk 启动。
- 支持稳定版通道；预发布版本只在用户主动选择测试通道后显示。
- 稳定版调用 `GET /repos/{owner}/{repository}/releases/latest`；测试通道调用 Releases 列表并筛选 `prerelease`。
- 使用 GitHub Releases API 返回的 `tag_name`、`published_at`、`body`、`html_url` 和 `assets` 生成更新结果。
- 发布仓库使用可配置的 `Owner/Repository`，正式发布前写入构建配置，不在代码中保存 GitHub Token。
- 公共仓库通过匿名 API 访问，并使用 `User-Agent`、`ETag` 和 `If-None-Match` 减少请求及避免触发速率限制。
- 本地缓存最近一次检查时间、`ETag` 和最新版本信息；网络不可用时保持静默，不影响桌面盒子运行。
- 使用语义化版本比较，忽略版本号前缀 `v`，不得使用普通字符串比较版本大小。
- Windows x64 安装资源采用固定命名，例如 `CrabDesk-Setup-x64.exe`，同时发布 `SHA256SUMS.txt`。
- 下载完成后必须校验 SHA-256；正式版还必须验证 Authenticode 签名和发布者信息。
- 发现新版本后只提示，不静默安装。下载或启动安装程序前必须获得用户确认。
- 安装更新时先正常退出 CrabDesk、恢复 Explorer 图标状态，再启动安装程序；不得直接覆盖正在运行的程序文件。
- 更新检查不上传配置、桌面文件列表、机器名、用户目录或其他使用数据。

应用内更新会在用户确认后下载固定名称的安装包和 `SHA256SUMS.txt`，完成哈希与签名验证，再正常退出 CrabDesk 并启动安装程序。稳定版强制要求可信 Authenticode 签名；未签名预览版需要二次确认。

GitHub 发布流程使用 `.github/workflows/release.yml`：

1. 推送格式为 `vX.Y.Z` 的标签后触发工作流。
2. 在 Windows Runner 上执行还原、Release 测试和 `win-x64` 自包含发布。
3. 使用 Inno Setup 生成 `CrabDesk-Setup-x64.exe`。
4. 生成 `SHA256SUMS.txt`，并在具备签名证书时完成 Authenticode 签名。
5. 创建 GitHub Release，上传安装包、校验文件和便携版压缩包。
6. 工作流只使用 GitHub Actions 提供的 `GITHUB_TOKEN` 发布资源，Token 不进入客户端程序或构建产物。

稳定版标签必须配置签名证书和密码；缺少任一 Secret 时工作流直接失败，不允许降级发布未签名稳定版。创建 Release 前会再次验证应用、IconGuard 和安装包均为受信任且带时间戳的 Authenticode 签名，并确认三个产物使用同一代码签名证书。预发布版本允许在明确标记为测试版时不签名。

### 3.8 关于

- 产品名称、版本、配置目录和开源许可。
- GitHub 仓库、发布通道和最近一次更新检查状态。

## 4. 核心数据模型

计划新增或扩展以下模型：

```text
OrganizationRule
  Id, Title, Enabled, Priority
  ItemKinds, NamePattern, Extensions
  TargetBoxId, Action

DesktopBehaviorSettings
  LaunchToTray, ToggleIconsOnDesktopDoubleClick
  ExpandBoxOnHover, RefreshAfterRename

GlobalAppearanceSettings
  CornerRadius, ShowBorder, ShowResizeGrip
  IconHorizontalSpacing, IconVerticalSpacing
  SelectionColor, HoverFeedback, AnimationEnabled

BoxAppearance
  Background, Accent, Opacity, TitleBarHeight
  IconSize, LabelFontSize, ShowItemLabels
  TitleAlignment, TitleColor, TitleFontSize, TitleFontBold

BackupSettings
  Enabled, RetentionDays, BackupDirectory
  LastBackupAt

MappedBoxSettings
  BoxId, FolderPath, IncludeSubdirectories
  ReadOnly

HotkeySettings
  ShowDesktop, OrganizeDesktop

UpdateSettings
  CheckOnStartup, Channel, LastCheckedAt
  CachedETag, LatestKnownVersion
```

配置架构每次只增加一个版本。迁移必须保留已有盒子、分组关系和用户主题设置。

## 5. 服务与接口

按现有项目边界补充以下服务：

- `IOrganizationRuleEngine`：规则匹配、预览和应用。
- `IDesktopChangeMonitor`：聚合文件系统、Shell 和 Explorer 变化事件。
- `IBackupService`：创建、枚举、导入、恢复和清理备份。
- `IMappedFolderProvider`：映射目录枚举与变化监听。
- `IHotkeyService`：注册、冲突检测和释放全局快捷键。
- `IThemeService`：系统主题监听、WinUI 资源和桌面盒子主题同步。
- `IUpdateService`：访问 GitHub Releases、缓存响应、比较版本、选择和下载发布资源并校验 SHA-256。

更新服务返回结构化结果，不直接操作设置窗口：

```text
UpdateCheckResult
  Status: UpToDate | UpdateAvailable | Offline | RateLimited | Failed
  CurrentVersion, LatestVersion, ReleaseName
  PublishedAt, ReleaseNotes, ReleasePageUrl
  InstallerUrl, Sha256Url, IsPrerelease
```

规则匹配和备份序列化放在 `CrabDesk.Core`；Shell、文件系统和快捷键实现放在 `CrabDesk.Native`；共享桌面运行时放在 `CrabDesk.Runtime`；设置页面与交互放在 `CrabDesk.WinUI`。

## 7. 测试矩阵

### 单元测试

- 规则优先级、通配符、扩展名和项目类型匹配。
- 分组操作不改变真实文件路径。
- 配置迁移、备份保留和损坏文件回退。
- 外观数值边界、盒子命中检测和多屏坐标换算。
- 卷序列号与文件 ID 在桌面文件重命名前后保持稳定。

### 集成测试

- Explorer 桌面宿主和原生图标定位。
- 未分组项目和原生桌面菜单始终由 Explorer 提供；已分组项目的原生图标停放到可视区域外，只在展开盒子内绘制。
- 文件新增、删除、重命名后的实时规则应用。
- 备份导入、恢复和失败回滚。
- Explorer 重启后的布局、规则和映射盒子恢复。
- 强制结束主进程后的 IconGuard 恢复。
- 全局快捷键注册、重复组合冲突和释放后重新注册。
- Shell 缩略图缓存命中、文件变化失效、容量淘汰和运行时内存/句柄增长。
- GitHub Releases 响应解析、`ETag` 缓存、限流和离线降级。
- 稳定版与预发布通道过滤、语义化版本比较和发布资源选择。
- 安装包 SHA-256 不匹配或签名无效时禁止启动安装。
- 盒子抗锯齿圆角与透明度屏幕像素、盒子范围窗口区域、收起后分组图标区域消失、重复 `Win+D` 与退出后的 Explorer 图标恢复。

### 人工验收

- 连续执行 `Win+D`、`Win+M` 和任务视图，盒子保持可见。
- 普通窗口、全屏应用和系统桌面菜单可以正常覆盖或操作。
- 设置窗口在最小尺寸和 `100%`、`125%`、`150%`、`200%` DPI 下无裁切。
- 深色、浅色、跟随系统模式下文字和控件对比度正确。
- 盒子在 `35%`、`50%` 和 `100%` 不透明度下边界清晰，壁纸正常透出且盒子外区域不受影响。
- 关闭设置窗口后进程驻留托盘，托盘退出后图标状态恢复。

## 9. 明确不做

- 不实现第三方软件的授权激活、代理设置和品牌内容。
- 不上传用户桌面文件或布局数据。
- 不在自动整理时移动真实文件，除非未来提供单独且明确确认的文件整理模式。
- 不支持 Windows 7/8、ARM64 或虚拟桌面独立布局，除非后续重新立项。
- 不为尚未实现的功能提前添加无效设置项。

## 10. 完成标准

单个功能只有同时满足以下条件才视为完成：

1. 设置项已持久化并能恢复。
2. 设置变化能立即作用于桌面或在界面中明确提示需要重启。
3. 异常退出不会破坏 Explorer 图标状态或配置文件。
4. 已覆盖关键单元测试或集成测试。
5. 深浅主题、窗口最小尺寸和高 DPI 下显示完整。
6. README 和本计划中的状态同步更新。
